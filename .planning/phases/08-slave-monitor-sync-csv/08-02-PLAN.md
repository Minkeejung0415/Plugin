# Plan 08-02: Compact Slave Monitor Panel in Acquisition Board Plugin

**Phase:** 08 - Slave Monitor, Sync Evidence, and CSV Export
**Status:** Planned
**Wave:** 2
**Depends on:** 08-01

## Objective

Add a fixed-height Slave Monitor panel to the existing Acquisition Board plugin so operators can select a slave from a dropdown and verify it is alive, SD-ready, recording, saving, sensor-responsive, synchronized, and producing quaternion values.

## Tasks

1. Extend board state to store parsed slave monitor records.
   - Add a small struct for slave identity, online/stale state, SD/recording fields, sensor fields, quaternion fields, DIO, sync, clock offset, saved samples, and errors.
   - Store records in a bounded collection keyed by MAC or slave ID.

2. Parse slave status lines from the master.
   - Support the machine-readable `SLAVE_STATUS` format from Plan 08-01.
   - Update slave records from the command/status socket without blocking acquisition.
   - Mark records stale if no update arrives within the configured age threshold.

3. Add a compact fixed-height UI panel.
   - Aggregate line: `Slaves 2/3 online  SD 2/3  REC 1/3  err 0`.
   - Dropdown/selector for detected slaves.
   - Selected-slave detail only: online age, SD, recording, saved samples, error count, accel/gyro/mag, quaternion, DIO, sync.
   - Do not create stacked cards or a variable-height list.

4. Make values readable at human-monitoring scale.
   - Convert raw accelerometer/mag/gyro/quaternion values to compact display where practical.
   - Show quaternion as normalized approximate floats or clearly label Q15 if not converted.
   - Use short labels and status colors/icons instead of verbose explanatory text.

5. Preserve existing Acquisition Board ergonomics.
   - Do not move or hide core acquisition controls.
   - Keep panel within current plugin height.
   - Avoid text overlap and long unbounded values.

6. Add operator status messages.
   - If no slaves are detected: show `Slaves: none`.
   - If slave stale/offline: show stale age.
   - If SD not ready or errors nonzero: show warning.
   - If recording but saved samples do not increase: show warning.

## Verification

- Plugin shows detected slave in dropdown within a few seconds of boot.
- Moving or rotating selected slave changes the displayed live values at about 5 Hz.
- RECORD start/stop updates selected slave recording state and saved sample count.
- UI remains fixed-height with 1, 2, and 4 detected slaves.
- Long MAC/session/error values do not overflow the panel.
- No Open Ephys acquisition thread blocking is introduced by status parsing.

## Success Criteria

- Operator can use only the Open Ephys plugin to verify a selected slave is alive and responding before recording.
- The plugin UI remains compact enough for the existing Acquisition Board editor.
