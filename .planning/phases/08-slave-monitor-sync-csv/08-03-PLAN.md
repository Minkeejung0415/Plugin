# Plan 08-03: Post-Acquisition CSV Conversion and Sync Evidence

**Phase:** 08 - Slave Monitor, Sync Evidence, and CSV Export
**Status:** Planned
**Wave:** 2
**Depends on:** 08-01

## Objective

Standardize the post-acquisition workflow for copied slave SD files: convert every `step_*.bin` to CSV, generate quality summaries, and make DIO/sync evidence easy to inspect.

## Tasks

1. Harden `sd_bin_to_csv.py`.
   - Keep direct `.bin` to `.csv` conversion.
   - Validate `STP1` header, version, channel count, and record size.
   - Preserve `seq,time_us,ax,ay,az,gx,gy,gz,mx,my,mz,qw,qx,qy,qz,dio` columns.
   - Add optional summary JSON output for single-file conversion.

2. Add `convert_sd_folder.py`.
   - Accept a folder or drive root.
   - Find `step_*.bin`.
   - Convert each file to `.csv`.
   - Generate per-file `*_summary.json`.
   - Generate a folder-level `conversion_summary.csv` or `.json`.

3. Add recording quality summary.
   - Record count, duration, first/last sequence, expected sample count, lost sequence count, duplicate sequence count, mean/median sample period.
   - Quaternion nonzero check and percentage of rows with nonzero quaternion.
   - DIO level changes/edge count and first/last edge times.
   - Basic SD debug file ingestion if `/relay_debug.txt` is present.

4. Add sync-analysis helpers.
   - For one file, report DIO edge times.
   - For multiple files in a folder, compare matching DIO edge indices across devices.
   - Estimate pairwise offset and drift from first/last repeated DIO pulses when available.
   - State clearly when there are not enough DIO edges to estimate sync.

5. Document the DIO test.
   - Explain `D0` pull-up behavior.
   - Single-board test: touch D0 to GND during recording and verify CSV `dio` changes.
   - Multi-board sync test: common signal to all D0 pins plus shared ground.
   - Warn against 5 V input.

6. Document the post-acquisition CSV workflow.
   - Binary on SD is the acquisition source of truth.
   - CSV is generated after acquisition.
   - Operator command examples for drive roots and output folders.

## Verification

- Running `python esp32/host/sd_bin_to_csv.py D:/step_x.bin` produces a CSV for a known valid file.
- Running `python esp32/host/convert_sd_folder.py D:/` converts all `step_*.bin` files.
- Summary reports correct record count and duration for a known file.
- Summary flags all-zero quaternion columns if slave firmware was flashed with filter disabled.
- Summary reports DIO edge count for a file with manual D0-to-GND pulses.
- Multi-file sync summary reports offset/drift when two or more files have matching DIO edge indices.

## Success Criteria

- After acquisition, the operator can convert all slave SD recordings to CSV with one command.
- The generated summary tells whether the file is continuous, has quaternion data, and contains usable DIO sync evidence.
