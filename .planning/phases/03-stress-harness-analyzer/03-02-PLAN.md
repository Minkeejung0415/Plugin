<objective>
Replace the always-zero host-row-index duplicate check with real payload-fingerprint duplicate
detection so that `dup_seq` in the results table reflects actual firmware-level duplicate samples.

Current behavior: `seq = len(rows)` is always 0,1,2,3... so the analyzer always reports
`dup_seq=0` regardless of whether the firmware re-broadcast the same sample.

Target behavior: compare each frame's 22-byte channel payload to the previous frame's payload.
Any match is a real duplicate (same IMU reading sent twice). Count and surface these.
</objective>
