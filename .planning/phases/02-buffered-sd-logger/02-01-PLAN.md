# Plan 02-01: ESP-IDF Buffered SD Logger

**Phase:** 02 - Buffered SD Logger
**Status:** Planned
**Requirements:** SD-01, SD-02, SD-03, SD-04, LOSS-01, STALL-01, STALL-02

## Scope

Implement a non-blocking SD logging path in the ESP-IDF firmware and wire acquisition-loop timing into logger-visible counters.

## Tasks

1. Extend `sd_logger.h` with a stats struct, append return value, stats getter, and acquisition timing hook.
2. Replace direct per-sample `fwrite()`/`fflush()` in `sd_logger.c` with a FreeRTOS queue and SD writer task.
3. Write a session header before `oe_sample_t` sample records.
4. Add plugin-facing `RECORD ON/OFF` firmware handling so Open Ephys controls SD save sessions.
5. Add Kconfig knobs for enabling SD logging, queue depth, flush interval, and log path.
6. Update `main.c` so the acquisition loop records generated samples, enqueue latency, loop duration, and overruns.
7. Add startup/runtime logs that make SD state and counters visible.

## Verification

- Code inspection confirms `sd_logger_append()` does not call file I/O or flush.
- Code inspection confirms the writer task is the only path that writes sample records.
- Code inspection confirms the Open Ephys command path can trigger `RECORD ON/OFF`.
- `git diff --check` passes for changed files.
- If ESP-IDF tools are available locally, build the firmware component/project.
