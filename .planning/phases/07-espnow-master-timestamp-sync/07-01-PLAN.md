# Plan 07-01: ESP-NOW Master Broadcast & Slave Timestamp Sync

**Phase:** 07 - ESP-NOW Master Broadcast & Slave Timestamp Sync
**Status:** Planned
**Wave:** 1

## Scope

Wire up the ESP-NOW skeleton that already exists in `step_node.ino` and `espnow_sync.c`.
The master (connected ESP) already calls `sendEspNowSync()` which broadcasts `{seq, time_us}`.
What is missing is:

1. The compile flag is off (`ENABLE_ESPNOW false`) — must be turned on.
2. The WiFi init path for USB+ESPNOW mode blocks 45 s trying to join a hotspot — must be
   bypassed with a fast STA-mode-only init.
3. `onEspNowRecv()` is an empty stub — must compute and store `clock_offset_us`.
4. `fillOeHeader()` and `logSd()` ignore the offset on slave nodes — must apply it.
5. `espnow_sync.c` stores master time but never computes `clock_offset_us` — must fix.
6. `main.c` never applies `clock_offset_us` on slave samples — must fix.

## Tasks

### A. Arduino sketch (step_node.ino) — primary path

**A-1. Enable ESP-NOW and add channel constant**

At the top of the file, change:
```
#define ENABLE_ESPNOW false
```
to:
```
#define ENABLE_ESPNOW true
#define ESPNOW_WIFI_CHANNEL 1   // All nodes must use the same channel
```

Rationale: The flag gates all ESP-NOW code paths. The channel must be the same on master
and all slaves. Channel 1 is the safest default (all 2.4 GHz devices support it).

**A-2. Add global sync state variables**

After the existing globals block (near `g_loop_overruns`, before `g_sd_path`), add:

```cpp
#if ENABLE_ESPNOW
static int64_t  g_clock_offset_us   = 0;
static bool     g_espnow_sync_received = false;
static uint32_t g_espnow_last_seq   = 0;
#endif
```

Rationale: The recv callback and timestamp functions need shared state. Keeping it
`#if ENABLE_ESPNOW` guarded avoids any overhead in single-node builds.

**A-3. Fix useWifi() to not require WiFi for ESPNOW-only mode**

Change:
```cpp
static bool useWifi() { return ENABLE_TCP || ENABLE_ESPNOW; }
```
to:
```cpp
static bool useWifi() { return ENABLE_TCP; }
```

Rationale: `useWifi()` gates the full `setupWifi()` which includes a 45-second STA timeout
scan. ESP-NOW needs WiFi initialized but NOT an AP connection. The WiFi init for ESP-NOW
will be handled inside `setupEspNow()` directly.

**A-4. Fix setupEspNow() — handle WiFi init and log success/failure**

Replace the existing `setupEspNow()` implementation with:

```cpp
static void setupEspNow() {
#if ENABLE_ESPNOW
  // When TCP is not in use, WiFi was not started by setupWifi().
  // ESP-NOW requires the WiFi stack to be initialized (STA mode, no AP join needed).
  if (!wifi_up) {
    WiFi.persistent(false);
    WiFi.disconnect(true);
    delay(100);
    WiFi.mode(WIFI_STA);
    WiFi.setSleep(false);
    // Force fixed channel so master and slaves agree without an AP
    esp_wifi_set_channel(ESPNOW_WIFI_CHANNEL, WIFI_SECOND_CHAN_NONE);
    delay(100);
    wifi_up = true;  // WiFi stack is up; no AP association needed for ESP-NOW
    Serial.printf("ESP-NOW WiFi: STA mode ch=%d (no AP join)\n", ESPNOW_WIFI_CHANNEL);
  }

  esp_err_t err = esp_now_init();
  if (err != ESP_OK) {
    Serial.printf("ESP-NOW init failed: %d\n", (int)err);
    return;
  }
  esp_now_register_recv_cb(onEspNowRecv);
  esp_now_register_send_cb(onEspNowSent);

  // Register broadcast peer (covers all slaves — no per-slave MAC needed)
  esp_now_peer_info_t peer = {};
  memset(peer.peer_addr, 0xFF, 6);
  peer.channel = 0;       // 0 = use current WiFi channel
  peer.encrypt = false;
  err = esp_now_add_peer(&peer);
  if (err != ESP_OK) {
    Serial.printf("ESP-NOW add peer failed: %d\n", (int)err);
    return;
  }
  Serial.printf("ESP-NOW ready (role=%s ch=%d)\n",
                NODE_IS_MASTER ? "master" : "slave", ESPNOW_WIFI_CHANNEL);
#else
  Serial.println("ESP-NOW disabled — single-node mode");
#endif
}
```

Note: `esp_wifi_set_channel()` requires including `esp_wifi.h` which is already present
in the file via `<WiFi.h>` (the Arduino ESP32 WiFi library exposes it), but add
`#include "esp_wifi.h"` after the existing `#include <WiFi.h>` line inside the
`#if ENABLE_ESPNOW` block to be explicit:
```cpp
#if ENABLE_ESPNOW
#include <esp_now.h>
#include "esp_wifi.h"
#endif
```

**A-5. Complete onEspNowRecv() — compute clock offset on slave**

Replace the existing stub:
```cpp
#if ENABLE_ESPNOW
void onEspNowRecv(const esp_now_recv_info_t *info, const uint8_t *data, int len) {
  (void)info; (void)data; (void)len;
}
```
with:
```cpp
#if ENABLE_ESPNOW
void onEspNowRecv(const esp_now_recv_info_t *info, const uint8_t *data, int len) {
  (void)info;
  if (NODE_IS_MASTER) return;  // master does not apply its own sync
  if (len < (int)sizeof(SyncPacket)) return;
  const SyncPacket *pkt = (const SyncPacket *)data;
  // Compute offset: how far ahead the master clock is relative to our local clock
  int64_t recv_us = (int64_t)esp_timer_get_time();
  g_clock_offset_us    = (int64_t)pkt->time_us - recv_us;
  g_espnow_sync_received = true;
  g_espnow_last_seq    = pkt->seq;
}
#endif
```

Rationale: The master's `time_us` was captured just before the packet was sent.
`recv_us` is captured as early as possible in the callback. The difference is the
master clock minus slave clock, which is the correction to apply to slave timestamps.
One-way latency bias (~100–300 µs at 1 m range) is acceptable for the 100 Hz use case.

**A-6. Apply offset in fillOeHeader()**

Change `fillOeHeader()` from:
```cpp
static void fillOeHeader(OeHeader *hdr) {
  const uint32_t hw_us = (uint32_t)esp_timer_get_time();
  hdr->offset = (int32_t)hw_us;
```
to:
```cpp
static void fillOeHeader(OeHeader *hdr) {
  int64_t t_us = (int64_t)esp_timer_get_time();
#if ENABLE_ESPNOW
  if (!NODE_IS_MASTER && g_espnow_sync_received) t_us += g_clock_offset_us;
#endif
  hdr->offset = (int32_t)(uint32_t)t_us;  // low 32 bits, same as before
```

**A-7. Apply offset in logSd()**

Change `logSd()` from:
```cpp
  rec.time_us = (int64_t)esp_timer_get_time();
```
to:
```cpp
  int64_t t_us = (int64_t)esp_timer_get_time();
#if ENABLE_ESPNOW
  if (!NODE_IS_MASTER && g_espnow_sync_received) t_us += g_clock_offset_us;
#endif
  rec.time_us = t_us;
```

**A-8. Add ESPNOW info to printAcqStatus()**

Inside `printAcqStatus()`, after the existing `Serial.printf("STATUS ...")` call, add:
```cpp
#if ENABLE_ESPNOW
  Serial.printf("ESPNOW role=%s ch=%d sync=%d offset_us=%lld last_seq=%lu\n",
                NODE_IS_MASTER ? "master" : "slave",
                ESPNOW_WIFI_CHANNEL,
                g_espnow_sync_received ? 1 : 0,
                (long long)g_clock_offset_us,
                (unsigned long)g_espnow_last_seq);
#endif
```

### B. ESP-IDF firmware (espnow_sync.c + main.c) — parity fix

**B-1. Compute clock_offset_us in espnow_recv callback**

In `espnow_sync.c`, change `espnow_recv()` from:
```c
    s_state.master_seq = p->seq;
    s_state.master_time_us = p->time_us;
```
to:
```c
    int64_t recv_us = esp_timer_get_time();
    s_state.master_seq     = p->seq;
    s_state.master_time_us = p->time_us;
    if (!s_master) {
        s_state.clock_offset_us = (int32_t)(p->time_us - recv_us);
    }
```

Rationale: Matches the Arduino logic. `clock_offset_us` is `int32_t` (fits ±35 min of drift,
more than sufficient for same-lab devices booting within seconds of each other).

**B-2. Apply clock_offset_us to slave sample.timestamp_us in main.c**

In `acquisition_task()`, change the existing ESP-NOW block:
```c
#if CONFIG_STEP_ENABLE_ESPNOW
        espnow_sync_on_local_frame(sample.seq, sample.timestamp_us);
#endif
```
to:
```c
#if CONFIG_STEP_ENABLE_ESPNOW
        if (!CONFIG_STEP_NODE_MASTER) {
            sync_state_t sync;
            espnow_sync_get_state(&sync);
            sample.timestamp_us += sync.clock_offset_us;
        }
        espnow_sync_on_local_frame(sample.seq, sample.timestamp_us);
#endif
```

Rationale: The offset is applied before `open_ephys_stream_set_sample()` and
`sd_logger_append()` so both paths use the corrected timestamp. On master,
`CONFIG_STEP_NODE_MASTER != 0` so the block is skipped.

## Notes

- **Channel lock**: `esp_wifi_set_channel(1, WIFI_SECOND_CHAN_NONE)` is called before `esp_now_init()`.
  Slaves must also use channel 1. If a slave is connected to a 2.4 GHz AP on a different channel,
  it must be reconfigured to channel 1 OR the AP must operate on channel 1.
- **Latency bias**: One-way ESP-NOW latency at 1–5 m indoor range is ~100–300 µs. This is not
  corrected in this phase (D-07). For 100 Hz (10 ms period) this is acceptable.
- **No download needed**: `esp_now.h`, `esp_wifi.h`, and `esp_timer.h` are already part of the
  ESP32 Arduino core and ESP-IDF SDK — no new dependencies required.
