# Plan 06-03: Plugin UI, Local Retrieval, and End-to-End Verification

**Phase:** 06 - Plugin-Controlled ESP32 SD Recording With Reconnect-Safe Local Retrieval
**Status:** Planned
**Wave:** 3
**Depends on:** 06-01, 06-02
**Requirements:** SD-05, LOSS-02, LOSS-03, LOSS-04, STALL-03, STALL-04, OPS-01, OPS-03

## Scope

Wire the Open Ephys plugin RECORD button to the new status/retrieval contract and verify that real operator runs save on ESP32 SD first, then retrieve a checked local copy.

## Tasks

1. Update ESP32 plugin RECORD start behavior so it sends the agreed start command and waits for firmware status before showing recording confirmed.
2. Update RECORD stop behavior so it requests firmware finalization, waits for flushed/closed confirmation, then retrieves the finalized file to a local path.
3. Replace misleading status text such as "Recording saved" with state-specific messages: command sent, recording confirmed, SD failure, finalizing, finalized, retrieving, retrieved, retrieval failed, timeout finalized.
4. Ensure the plugin no longer silently uses the legacy ESP32 PC-side CSV path for normal recording.
5. Add safe retry behavior: if retrieval fails after SD finalization, preserve the SD file path/session id and allow the operator to retry retrieval without restarting acquisition.
6. Add local retrieved-file verification by invoking or documenting the existing SD analyzer path.
7. Update operator documentation for the final workflow: insert SD, start plugin bridge if needed, press RECORD, stop RECORD, wait for retrieval, run analyzer or review pass/fail result.

## Verification

- No-SD dry run: plugin shows truthful SD failure instead of claiming a saved recording.
- SD smoke run: plugin starts SD recording, stop finalizes it, local retrieval completes, and firmware reports `sd_saved > 0` with `sd_errors = 0`.
- Retrieved file passes `esp32/host/analyze_sample_rate.py --format sd-bin`.
- Connection-loss run: disconnect during recording, reconnect before timeout, stop from plugin, retrieve file, and verify continuity.
- Timeout run: disconnect beyond grace period, reconnect, retrieve finalized file, and verify status says timeout-finalized.
- Open Ephys/plugin remains responsive during finalize/retrieval operations.
