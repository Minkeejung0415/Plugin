# Plan 06-02: Firmware and Bridge Recording Control

**Phase:** 06 - Plugin-Controlled ESP32 SD Recording With Reconnect-Safe Local Retrieval
**Status:** Planned
**Wave:** 2
**Depends on:** 06-01
**Requirements:** SD-01, SD-02, SD-03, SD-04, SD-05, LOSS-01, LOSS-03, LOSS-04, STALL-01, STALL-02, STALL-03

## Scope

Implement firmware and bridge support so plugin RECORD commands actually start/stop ESP32 SD logging, survive temporary host disconnects, and expose truthful status.

## Tasks

1. Add a top-of-file configurable reconnect grace constant to the relevant firmware path, defaulting to a one-to-two-minute value.
2. Extend firmware recording state so SD logging can continue when the host command connection drops.
3. On host disconnect during recording, start the grace timer and keep appending samples to SD.
4. On reconnect before timeout, cancel the grace timer and continue the existing recording session without a false stop/start boundary.
5. On timeout, flush/close the SD file, stop logging, record finalization reason as disconnect timeout, and preserve final status for the next host connection.
6. Add or extend firmware status commands so direct TCP and/or serial paths can report recording state, counters, finalization reason, and file metadata.
7. Add file retrieval commands or a transfer endpoint that can read the finalized SD file after recording stops.
8. Update `esp32/host/serial_tcp_bridge.py` so USB bridge mode relays `RECORD` and related status/retrieval commands, not just `FREQ`, `CFG`, `FILTER`, and `STOP`.
9. Ensure file-transfer framing cannot be mistaken for live Open Ephys sample frames.

## Verification

- Code inspection confirms SD logging continues after host disconnect while the grace timer is active.
- Reconnect-before-timeout test confirms the session continues and `sd_saved` keeps increasing.
- Timeout test confirms the file is flushed/closed and status reports timeout finalization.
- USB bridge test confirms `RECORD ON/OFF` reaches Arduino firmware and status returns.
- Retrieval test confirms a finalized SD file can be copied from ESP32 to local storage without corrupting the SD source file.
- `git diff --check` passes for changed files.
