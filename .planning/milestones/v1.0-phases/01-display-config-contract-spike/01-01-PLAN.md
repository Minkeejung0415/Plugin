<action>
Create `docs/opensim-joint-display-config.md` documenting:

1. **File location:** `{WORK_DIR}/opensim_joint_display_config.json` where WORK_DIR matches `opensim_live_realtime.py` line 50 and `kOpenSimWorkDir` in `acqboard.ccp`.

2. **Schema fields:**
   - `joints`: array of OpenSim coordinate name strings from curated catalog; max 6; order preserved for display
   - `trigger_ts`: float, Unix epoch seconds when config was applied
   - `seq`: integer, monotonic sequence; readers ignore stale seq

3. **Example JSON** with 2 joints: `["knee_angle_r", "hip_flexion_r"]`

4. **Locked defaults table:** curated catalog only, apply-on-trigger semantics (Phase 3), max 6 joints, primary flexion only, display format `knee_r: 42.1°`

5. **Atomic write pattern** (for Phase 3): write `.tmp` then rename — document but do not implement C++ writer in this plan

6. **Cross-reference** `docs/opensim-udp-v2.md` stating display config is separate from UDP v2 packets

7. **Sensor vs joint distinction:** IMU segment names (`femur_r_imu`) vs coordinate names (`hip_flexion_r`)
</action>

- Document states max 6 joints and abbreviated display format with 1 decimal degree

<read_first>

</read_first>

<action>
Create `opensim_joint_catalog.py` at repo root (same directory as `opensim_live_realtime.py`) with:

1. `MAX_DISPLAY_JOINTS = 6`

2. `JOINT_CATALOG`: ordered dict or list of entries, each with:
   - `coordinate`: OpenSim name (e.g. `knee_angle_r`)
   - `abbrev`: display label (e.g. `knee_r`)
   - `segment`: IMU segment hint for Phase 2 UI grouping (e.g. `tibia_r_imu`)

3. Include exactly these primary flexion coordinates:
   `pelvis_tilt`, `hip_flexion_r`, `knee_angle_r`, `ankle_angle_r`, `hip_flexion_l`, `knee_angle_l`, `ankle_angle_l`

4. Functions:
   - `abbrev_for(coordinate: str) -> str` — returns abbrev or coordinate if unknown
   - `validate_joints(joints: list) -> list` — filters to catalog-known names, truncates to 6, logs would-be drops via return metadata or raises no exception (caller logs)
   - `format_angle_line(abbrev: str, degrees: float) -> str` — returns `f"{abbrev}: {degrees:.1f}°"`

5. No OpenSim import in this module (pure Python for easy unit testing)
</action>

- `JOINT_CATALOG` includes `knee_angle_r` with abbrev `knee_r`
- `format_angle_line("knee_r", 42.123)` returns string matching pattern `knee_r: 42.1°`
- `validate_joints` with 8 valid names returns at most 6 entries
</acceptance_criteria>
