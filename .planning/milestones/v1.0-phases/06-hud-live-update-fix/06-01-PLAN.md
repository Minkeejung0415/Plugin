<objective>
Fix the frozen joint-angle HUD (DISP-02). The Phase-4 readout is permanently stuck on
`knee_r: --.--°` because `addDecoration` stores `DecorativeText` by value and the per-frame
`setText` mutates a stale copy the renderer never reads. Phase-6 research confirmed the locked
"DecorationGenerator" fix is unbuildable on this OpenSim 4.5 build (the class is unwrapped) and
that NO in-Simbody-viewport live text is possible. This plan implements the operator-selected
LAYERED strategy: a real runtime capability probe that selects `window_title` (revived via a
SWIG-wrapped `SimTK.String`) and falls through to the already-wired port-5001 `udp_feedback`,
gating the dead `screen_text` path so it can never silently re-freeze. The fix lands once in the
source-of-truth copy and is then made byte-identical in the executed copy.

Purpose: Close DISP-02 — give the operator a readout that verifiably updates each frame.
Output: Edited `opensim_live_realtime.py` (both copies, identical) with `_probe_hud_capabilities`,
a rewritten `_pick_hud_strategy`, a wrapped-String window-title attempt, and a gated screen_text path.
</objective>
